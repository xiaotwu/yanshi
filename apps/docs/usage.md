# Usage

Yanshi is chat-first. You describe the work, then the runtime plans, uses tools, asks for approval
when needed, and returns files or outputs.

## Chats

Use **New Chat** for standalone work. Use a project-scoped chat when the work should share files,
history, and settings with a project.

## Projects

Projects group chats, files, outputs, agent configuration, and Yanshi Atelier state.

## Tools

Yanshi can use file, browser, computer, terminal, and Docker-backed tools. Tools only run when the
required local dependency or macOS permission is available.

## Approvals

Risky actions pause for approval. Approve or deny the request, and Yanshi resumes from that real
decision.

## Library

Generated files and outputs appear in the chat and Library with real names and paths.

## Settings

Use Settings for providers, API keys, permissions, appearance, shortcuts, and Developer Mode.
