# Yanshi Continuation Protocol

This project is expected to take multiple Codex sessions.

Codex must maintain these files:

## IMPLEMENTATION_PLAN.md

Contains:
- Full milestone plan
- Current milestone
- Completed milestones
- Pending milestones
- Acceptance criteria

## IMPLEMENTATION_LOG.md

Append-only log.

Each entry should include:
- Date/time
- Phase
- Files changed
- Commands run
- Results
- Issues found
- Next action

## CURRENT_STATUS.md

Short summary for next Codex session.

Must include:
- What works
- What does not work
- Current failing tests
- Current branch
- Current app start command
- Current runtime start command
- Current blockers

## NEXT_STEPS.md

Ordered next steps.

Use clear checkboxes.

## ACCEPTANCE_CHECKLIST.md

Full no-mock and feature checklist.

Keep updated.

## Rule

When usage runs out, the next Codex session should be able to continue by reading only:

- AGENTS.md
- skills
- design spec
- implementation files above
